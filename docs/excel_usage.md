# Excel 文件转换使用指南

## 概述

本指南说明如何使用实验室智能助手系统处理 Excel 文件。系统使用 Microsoft MarkItDown 库将 Excel 文件转换为 Markdown 格式，然后上传到知识库。

## 支持的格式

- `.xlsx` - Excel 2007 及以上版本
- `.xls` - Excel 97-2003 版本

## 转换特性

### MarkItDown 转换特点

1. **表格结构保留**: 完整保留 Excel 的行列结构和表头
2. **多工作表支持**: 自动处理多个工作表，每个工作表转换为独立的 Markdown 部分
3. **格式兼容**: 处理合并单元格、数字、日期等格式
4. **文本质量**: 输出的 Markdown 文本适合 LLM 处理和分析

### 备选方案

如果 MarkItDown 不可用，系统会使用 openpyxl 作为备选方案：
- 保留表格结构
- 支持多工作表
- 格式处理略有差异

## 使用方法

### 方式 1: 通过 Web 界面上传

1. 访问知识库管理页面
2. 点击「选择文件」按钮
3. 选择 `.xlsx` 或 `.xls` 文件
4. 点击「上传」按钮
5. 系统自动转换为 Markdown 并添加到知识库

### 方式 2: 通过 API 上传

```bash
curl -X POST "http://localhost:8000/api/upload" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "files=@your_file.xlsx"
```

### 方式 3: 命令行转换

```bash
cd backend
python scripts/excel2markdown.py path/to/your_file.xlsx [output_directory]
```

## 转换示例

### 输入 Excel 文件

| 姓名 | 年龄 | 职位 |

| 张三 | 28 | 工程师 |
| 李四 | 35 | 经理 |

| 张三 | 28 | 工程师 |
| 李四 | 35 | 经理 |

## 最佳实践

### 1. 文件准备

- 确保 Excel 文件格式正确
- 删除不必要的空行和空列
- 保持表头清晰明确
- 重要数据放在前几个工作表中

### 2. 格式建议

**推荐**:
- 使用简单的表格结构
- 保持数据类型一致
- 避免复杂的合并单元格

**不推荐**:
- 包含大量公式的表格（可能无法正确提取值）
- 嵌套表格结构
- 包含宏或 VBA 代码的工作簿

### 3. 大文件处理

- 单个文件不建议超过 100MB
- 如果文件很大，系统会自动分块处理
- 建议将大型 Excel 拆分为多个小文件

## 故障排除

### 问题 1: 转换失败

**症状**: 上传 Excel 文件后提示转换失败

**解决方案**:
1. 检查是否安装了必要的库:
   ```bash
   pip install 'markitdown[xlsx]'
   ```
2. 确认 Excel 文件没有损坏
3. 尝试用 Excel/WPS 重新保存文件
4. 查看后端日志获取详细错误信息

### 问题 2: 内容不完整

**症状**: 转换后的 Markdown 缺少部分内容

**解决方案**:
1. 检查 Excel 文件是否有隐藏的工作表
2. 确认没有保护的单元格
3. 检查是否有不支持的格式（如特殊的图表）
4. 尝试使用 MarkItDown 的最新版本

### 问题 3: 格式错乱

**症状**: Markdown 表格格式不正确

**解决方案**:
1. 确保表头行清晰
2. 避免使用合并单元格
3. 检查是否有多余的空行
4. 确认数据区域的连续性

## 技术细节

### 转换流程

1. **文件读取**: 系统读取 Excel 文件
2. **工作表遍历**: 遍历所有工作表
3. **数据提取**: 提取每个单元格的文本值
4. **格式转换**: 将数据转换为 Markdown 表格格式
5. **知识库存储**: 将 Markdown 内容分块并向量化存储

### 单元格处理规则

| Excel 数据类型 | 处理方式 |
|--------------|---------|
| 文本 | 直接保留 |
| 数字 | 转换为字符串 |
| 日期 | 保持日期格式 |
| 空值 | 保留为空 |
| 公式 | 提取计算后的值（如果可能）|

### 特殊字符转义

Markdown 表格中的管道符 `|` 会被转义为 `\|`，确保表格结构正确。

## 性能考虑

### 转换时间

| 文件大小 | 预计转换时间 |
|---------|------------|
| < 1MB | < 1秒 |
| 1-10MB | 1-5秒 |
| 10-50MB | 5-30秒 |
| > 50MB | > 30秒 |

### 内存使用

- 小文件（< 10MB）: < 100MB 内存
- 中等文件（10-50MB）: 100-500MB 内存
- 大文件（> 50MB）: > 500MB 内存

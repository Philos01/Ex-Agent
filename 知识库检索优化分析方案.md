# 知识库检索功能优化分析方案

## 一、现有检索机制局限性分析

### 1.1 系统架构概述
当前系统采用混合检索架构：
- BM2.5 关键词检索 + 向量语义检索并行执行
- 使用 Reciprocal Rank Fusion (RRF) 算法融合结果
- 基于 BAAI/bge-reranker-v2-m3 模型进行重排序
- 默认返回 top_k=3 个文档片段

### 1.2 核心局限性

#### 1.2.1 检索结果数量限制
- 默认仅返回 3 个文档片段，无法覆盖多文档需求
- 配置参数 `final_select_count` 固定为 3，缺乏动态调整机制

#### 1.2.2 相关性评分机制单一
- 重排序仅考虑查询与单个文档片段的相关性
- 未考虑文档间的互补性、关联性和覆盖度
- 缺少文档多样性评估指标

#### 1.2.3 缺乏多文档信息融合策略
- 检索结果按简单排序呈现，无结构化整合
- 未对同一文档的多个片段进行聚合
- 缺少跨文档信息的关联性分析

#### 1.2.4 查询理解能力有限
- 仅进行单一查询改写，无子问题分解
- 无法识别需要多文档综合回答的复杂问题
- 缺少多跳检索机制

#### 1.2.5 上下文构建策略简单
- 所有检索到的文档片段简单拼接
- 未按文档来源或主题进行组织
- 缺乏对冗余信息的过滤

## 二、优化方案总体架构

### 2.1 设计目标
- 支持跨多篇文章内容的综合检索
- 提高复杂问题的回答准确性和完整性
- 保持系统响应性能
- 增强检索结果的多样性和覆盖度

### 2.2 优化架构
```
用户查询
    ↓
[查询分析与分解] → 子问题生成
    ↓
[多策略检索] → BM2.5 + 向量 + 多跳检索
    ↓
[文档多样性重排序] → 相关性 + 多样性 + 覆盖度
    ↓
[多文档信息融合] → 文档聚合 + 主题聚类 + 冗余过滤
    ↓
[上下文结构化构建] → 按文档/主题组织
    ↓
LLM 生成回答
```

## 三、具体技术措施

### 3.1 检索算法改进

#### 3.1.1 子问题分解 (Query Decomposition)
**实现思路**：
- 使用 LLM 将复杂问题分解为多个子问题

#### 3.1.2 多跳检索 (Multi-hop Retrieval)
**实现思路**：
- 第一轮：检索初始相关文档
- 第二轮：基于第一轮检索结果中的关键实体/概念进行二次检索
- 支持多轮迭代，直到覆盖足够信息

**关键实现**：
```python
class MultiHopRetriever:
    def multi_hop_search(self, query: str, hops: int = 2) -> List[Dict]:
        all_docs = []
        current_query = query
        
        for i in range(hops):
            docs = hybrid_search(current_query, initial_count=30, final_count=10)
            all_docs.extend(docs)
            
            if i < hops - 1:
                # 从当前文档中提取关键概念生成新查询
                current_query = self._extract_next_query(docs, query)
        
        return self._deduplicate(all_docs)
```

#### 3.1.3 动态检索数量调整
**实现思路**：
- 根据问题复杂度动态调整检索数量
- 简单问题：返回 3-5 个文档
- 复杂问题：返回 8-15 个文档
- 基于查询长度、实体数量等特征评估复杂度

### 3.2 相关性评分机制调整

#### 3.2.1 多维度评分框架
**评分维度**：
1. **语义相关性** (40%)：查询与文档的语义相似度
2. **关键词匹配度** (25%)：BM2.5 分数
3. **文档多样性** (20%)：与已选文档的差异性
4. **信息覆盖度** (15%)：对查询不同方面的覆盖

#### 3.2.2 多样性增强重排序
**实现思路**：
- 使用 Maximal Marginal Relevance (MMR) 算法
- 平衡相关性和多样性
- 避免返回内容高度相似的文档

**关键实现**：
```python
def mmr_rerank(
    query: str, 
    docs: List[Dict], 
    top_k: int = 10, 
    lambda_param: float = 0.7
) -> List[Dict]:
    selected = []
    candidates = docs.copy()
    
    while len(selected) < top_k and candidates:
        # 计算每个候选文档的 MMR 分数
        best_doc = None
        best_score = -1
        
        for doc in candidates:
            relevance = doc["score"]
            diversity = 1 - max([
                cosine_similarity(doc["embedding"], s["embedding"]) 
                for s in selected
            ]) if selected else 1
            
            mmr_score = lambda_param * relevance + (1 - lambda_param) * diversity
            
            if mmr_score > best_score:
                best_score = mmr_score
                best_doc = doc
        
        if best_doc:
            selected.append(best_doc)
            candidates.remove(best_doc)
    
    return selected
```

#### 3.2.3 文档来源分布优化
- 确保检索结果来自多个不同文档
- 避免单个文档垄断检索结果
- 设置单文档最大片段数限制（如 3-5 个）

### 3.3 多文档信息融合策略

#### 3.3.1 文档级别聚合
**实现思路**：
- 将同一文档的多个片段聚合
- 保留文档元数据（来源、时间等）
- 构建文档级别的上下文摘要

**关键实现**：
```python
def aggregate_by_document(docs: List[Dict]) -> Dict[str, List[Dict]]:
    doc_groups = defaultdict(list)
    
    for doc in docs:
        source = doc["metadata"].get("source", "unknown")
        doc_groups[source].append(doc)
    
    # 对每个文档的片段按位置排序
    for source in doc_groups:
        doc_groups[source].sort(
            key=lambda x: x["metadata"].get("chunk_index", 0)
        )
    
    return doc_groups
```

#### 3.3.2 主题聚类 (Topic Clustering)
**实现思路**：
- 使用嵌入向量对检索结果进行聚类
- 识别不同的主题/观点
- 每个主题保留代表性文档

**关键实现**：
```python
from sklearn.cluster import KMeans

def cluster_documents(docs: List[Dict], n_clusters: int = None) -> Dict[int, List[Dict]]:
    if not docs:
        return {}
    
    embeddings = np.array([doc["embedding"] for doc in docs])
    
    if n_clusters is None:
        n_clusters = min(len(docs) // 2, 5)
        n_clusters = max(n_clusters, 2)
    
    kmeans = KMeans(n_clusters=n_clusters, random_state=42)
    clusters = kmeans.fit_predict(embeddings)
    
    result = defaultdict(list)
    for doc, cluster_id in zip(docs, clusters):
        result[cluster_id].append(doc)
    
    return result
```

#### 3.3.3 冗余信息过滤
**实现思路**：
- 检测并去除高度相似的文档片段
- 保留信息含量更高的片段
- 使用相似度阈值进行过滤

### 3.4 上下文结构化构建

#### 3.4.1 按文档组织上下文
```
【文档1：论文标题A.pdf】
片段1：...
片段2：...

【文档2：论文标题B.pdf】
片段1：...
```

#### 3.4.2 增强的 Prompt 设计
**关键改进**：
- 明确告知 LLM 可以综合多个文档的信息
- 要求 LLM 标注信息来源（文档名称）
- 提供文档间对比的指引

**Prompt 示例**：
```
你是遥感图像融合领域的专业智能助手。请根据以下多个文档的内容综合回答问题。

### 参考文档：
{structured_context}

### 当前问题：
{question}

### 回答要求：
1. 综合多个文档的信息，不要局限于单一文档
2. 每个关键结论标注来源，格式：[文档名]
3. 如果不同文档存在观点差异，请明确指出
4. 保持学术严谨性和专业术语准确性
```

## 六、验证方法

### 6.1 测试数据集构建
1. **单文档问题集**（50题）：答案仅需单一文档
2. **双文档问题集**（50题）：答案需要综合两个文档
3. **多文档问题集**（30题）：答案需要综合三个及以上文档
4. **开放域复杂问题**（20题）：真实场景的复杂问题

### 6.2 评估指标

#### 6.2.1 检索质量指标
- **Recall@k**：检索到相关文档的比例
- **Precision@k**：检索结果中相关文档的比例
- **NDCG**：归一化折损累计增益
- **文档覆盖率**：检索结果涉及的不同文档数量

#### 6.2.2 回答质量指标
- **准确性**：人工标注答案的正确性
- **完整性**：答案覆盖问题各方面的程度
- **引用质量**：引用来源的准确性和相关性
- **连贯性**：答案的逻辑连贯性

### 6.3 A/B 测试方案
1. **分组**：50% 用户使用优化版，50% 使用原版
2. **时长**：2-4 周
3. **关键指标**：
   - 用户满意度评分
   - 对话轮数（是否需要追问）
   - 回答接受率

### 6.4 消融实验
验证各模块的贡献：
- 移除子问题分解
- 移除多样性重排序
- 移除多跳检索
- 对比完整系统与各变体的性能

## 七、风险与应对

### 7.1 性能风险
- **风险**：检索复杂度增加导致响应变慢
- **应对**：实现缓存机制、异步处理、结果预取

### 7.2 成本风险
- **风险**：LLM 调用次数增加导致成本上升
- **应对**：优化查询分解策略、限制跳数、使用轻量级模型

### 7.3 质量风险
- **风险**：子问题分解错误导致检索偏离
- **应对**：添加质量校验、保留原始查询检索结果、人工审核

## 八、总结

本优化方案通过以下关键改进实现跨多文档综合检索：
1. **查询理解增强**：子问题分解 + 多跳检索
2. **检索结果优化**：多样性重排序 + 动态数量调整
3. **信息融合**：文档聚合 + 主题聚类 + 冗余过滤
4. **上下文构建**：结构化组织 + 增强的 Prompt

通过分阶段实施和严谨验证，可以在保持系统性能的同时，显著提升复杂问题的回答质量。

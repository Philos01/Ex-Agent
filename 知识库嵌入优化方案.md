# 知识库嵌入优化方案：父文档检索（Parent-Document Retrieval）

> **版本**: v2.0  
> **更新日期**: 2026-04-28  
> **适用项目**: 实验室Agent (RS-NBU 课题组学术助手)  
> **技术栈**: Python 3.8+ / FastAPI / ChromaDB / Sentence-Transformers / SQLite

---

## 一、方案概述

### 1.1 核心问题

当前系统的文档切分策略采用**扁平化分块**（[ingest.py:125-233](backend/app/services/ingest.py#L125-L233)）：将文档按章节拆分后直接合并为固定大小的 chunk（chunk_size=1500, overlap=225），存入 ChromaDB 向量库。检索时直接返回匹配的 chunk 片段。

**存在的问题**：
- **语义断裂**：一个完整章节被切到不同 chunk 中，LLM 只能看到碎片化的上下文
- **精准度与完整度矛盾**：chunk 太小则语义不完整，chunk 太大则检索精准度下降
- **引用困难**：无法精确定位到"第 3.2 节 第二段"这样的粒度

### 1.2 解决方案：父文档检索（Parent-Document Retrieval）

**核心思想**：将 Markdown 结构化拆分和父文档检索结合——一个完整章节作为"父文档"（保证上下文完整），再切分成小粒度的"子切片"（保证检索精准）。检索时用子切片匹配，返回整个父文档。

```
┌─────────────────────────────────────────────────────────────────┐
│                    父文档检索架构 (Parent-Document Retrieval)      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────┐    ┌──────────────┐    ┌──────────────────────┐  │
│  │ 原始文档  │───▶│ Markdown 章节 │───▶│ 父文档 (Parent)       │  │
│  │ (MD/PDF) │    │ 感知拆分      │    │ 完整章节 + 标题元数据   │  │
│  └──────────┘    └──────────────┘    └────────┬─────────────┘  │
│                                               │                │
│                    ┌──────────────────────────┼──────────┐     │
│                    │                          │           │     │
│                    ▼                          ▼           │     │
│  ┌──────────────────────┐    ┌──────────────────────┐    │     │
│  │ 子切片 (Child Chunks) │    │ 父文档存储 (SQLite)    │    │     │
│  │ 200-500 字符/片       │    │ parent_id → 完整内容   │    │     │
│  │ ChromaDB 向量库       │    │ + 章节标题 + 文件名     │    │     │
│  └──────────┬───────────┘    └───────────┬──────────┘    │     │
│             │                            │                │     │
│             └──────────┬─────────────────┘                │     │
│                        │                                  │     │
│                        ▼                                  │     │
│  ┌──────────────────────────────────────────────────────┐ │     │
│  │                    检索流程                            │ │     │
│  │  用户查询 → 子切片匹配(top-k) → 去重parent_id          │ │     │
│  │  → 提取完整父文档 → 拼接上下文 → LLM 生成回答           │ │     │
│  └──────────────────────────────────────────────────────┘ │     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、技术方案详述

### 2.1 双层切分策略

#### 2.1.1 第一层：章节感知父文档生成

**目标**：将文档按自然章节边界拆分为"父文档"，每个父文档是一个语义完整的逻辑单元。

**复用现有基础**：项目已有 `_split_by_sections()` 函数（[ingest.py:145-193](backend/app/services/ingest.py#L145-L193)），按英文/中文学术论文章节标题识别章节边界。需在此基础上增强：

```
增强点:
1. 增加 Markdown #/##/### 标题识别（优先级最高）
2. 父文档最小长度阈值：min_parent_chars=300（过短章节合并至相邻章节）
3. 父文档最大长度阈值：max_parent_chars=8000（过长章节按二级标题再次拆分）
4. 保留完整的章节层级信息到 metadata
```

**实现代码框架**：

```python
# backend/app/services/parent_document.py (新文件)

import re
import uuid
from typing import List, Dict, Tuple
from dataclasses import dataclass, field

@dataclass
class ParentDocument:
    """父文档数据结构"""
    parent_id: str          # 唯一标识 UUID
    title: str              # 章节标题（如 "3.2 实验方法"）
    title_hierarchy: List[str]  # 标题层级 ["第三章", "3.2 实验方法"]
    content: str            # 完整章节内容
    filename: str           # 来源文件名
    section_level: int      # 章节层级 (1=#, 2=##, 3=###)
    char_count: int         # 字符数
    child_chunk_ids: List[str] = field(default_factory=list)  # 关联的子切片ID

def generate_parent_documents(
    text: str,
    filename: str,
    min_parent_chars: int = 300,
    max_parent_chars: int = 8000
) -> List[ParentDocument]:
    """
    第一层：生成父文档

    策略：
    1. 优先按 Markdown #/##/### 标题拆分
    2. 对于无标题的纯文本，按空行（段落边界）拆分
    3. 过短章节合并至前一个章节
    4. 过长章节按二级/三级标题递归拆分
    """
    lines = text.split('\n')
    parents = []
    current_lines = []
    current_title = ""
    current_hierarchy = []
    current_level = 0

    # 增强的 Markdown 标题正则（支持中英文混排）
    md_header_pattern = re.compile(
        r'^(#{1,4})\s+(.+?)(?:\s*\{#[\w-]+\})?\s*$'  # 支持 {#custom-id}
    )

    # 英文学术论文章节模式（保留原有逻辑）
    en_section_pattern = re.compile(
        r'^\s*(\d+\.?\s+[A-Z][a-zA-Z\s,:\-]+)',
        re.MULTILINE
    )

    # 中文学术论文章节模式（保留原有逻辑）
    cn_section_pattern = re.compile(
        r'^\s*(摘要|引言|相关工作|方法|实验|结果|讨论|结论|参考文献|致谢|附录)',
        re.MULTILINE
    )

    def flush_parent():
        """将当前累积的内容输出为一个父文档"""
        nonlocal current_lines, current_title, current_hierarchy, current_level
        content = '\n'.join(current_lines).strip()
        if not content or len(content) < 20:
            current_lines = []
            return

        parent = ParentDocument(
            parent_id=str(uuid.uuid4()),
            title=current_title or "无标题",
            title_hierarchy=current_hierarchy.copy(),
            content=content,
            filename=filename,
            section_level=current_level,
            char_count=len(content)
        )
        parents.append(parent)
        current_lines = []

    # 主循环：逐行扫描
    for line in lines:
        md_match = md_header_pattern.match(line)
        en_match = en_section_pattern.match(line)
        cn_match = cn_section_pattern.match(line)

        is_section_header = False
        header_text = ""
        header_level = 0

        if md_match:
            is_section_header = True
            header_level = len(md_match.group(1))
            header_text = md_match.group(2).strip()
        elif en_match and len(line.strip()) < 100:
            is_section_header = True
            header_level = 2
            header_text = en_match.group(1).strip()
        elif cn_match and len(line.strip()) < 100:
            is_section_header = True
            header_level = 2
            header_text = cn_match.group(1).strip()

        if is_section_header:
            # 输出前一个章节
            flush_parent()

            # 开始新章节
            current_title = header_text
            current_level = header_level
            # 构建层级路径
            if header_level == 1:
                current_hierarchy = [header_text]
            elif header_level == 2:
                current_hierarchy = [current_hierarchy[0]] if current_hierarchy else []
                current_hierarchy.append(header_text)
            else:
                current_hierarchy.append(header_text)
            current_lines = [line]
        else:
            current_lines.append(line)

    # 输出最后一个章节
    flush_parent()

    # 后处理：合并过短的父文档
    parents = _merge_short_parents(parents, min_parent_chars)

    # 后处理：拆分过长的父文档
    parents = _split_long_parents(parents, max_parent_chars)

    logger.info(f"父文档生成完成: {filename} → {len(parents)} 个父文档")
    return parents


def _merge_short_parents(
    parents: List[ParentDocument],
    min_chars: int
) -> List[ParentDocument]:
    """合并过短的父文档到前一个"""
    merged = []
    for p in parents:
        if merged and p.char_count < min_chars:
            # 合并到前一个
            merged[-1].content += '\n\n' + p.content
            merged[-1].char_count += p.char_count
            merged[-1].title_hierarchy.extend(p.title_hierarchy)
            merged[-1].child_chunk_ids.extend(p.child_chunk_ids)
        else:
            merged.append(p)
    return merged


def _split_long_parents(
    parents: List[ParentDocument],
    max_chars: int
) -> List[ParentDocument]:
    """按段落边界拆分过长的父文档"""
    result = []
    for p in parents:
        if p.char_count <= max_chars:
            result.append(p)
            continue

        # 按空行（段落边界）拆分
        paragraphs = p.content.split('\n\n')
        sub_parents = []
        current_text = ""
        part_num = 1

        for para in paragraphs:
            if len(current_text) + len(para) < max_chars:
                current_text = (current_text + '\n\n' + para).strip()
            else:
                if current_text:
                    sub_parents.append(_create_sub_parent(p, current_text, part_num))
                    part_num += 1
                current_text = para

        if current_text:
            sub_parents.append(_create_sub_parent(p, current_text, part_num))

        result.extend(sub_parents)
    return result
```

#### 2.1.2 第二层：子切片生成

**目标**：将每个父文档切分为小而精准的子切片，用于向量检索匹配。

**设计参数**：

| 参数 | 值 | 依据 |
|------|-----|------|
| `child_chunk_size` | 300 字符 | 约 150 中文词，足够表达一个完整观点 |
| `child_chunk_overlap` | 60 字符 | 20% 重叠率，确保边界语义不丢失 |
| 切分策略 | `RecursiveCharacterTextSplitter` | 优先按句号/换行切分，保证句子完整 |

**实现代码框架**：

```python
def generate_child_chunks(
    parent: ParentDocument,
    child_chunk_size: int = 300,
    child_chunk_overlap: int = 60
) -> List[Dict]:
    """
    第二层：从父文档生成子切片

    每个子切片携带 parent_id 用于回溯检索。
    """
    import re

    text = parent.content
    children = []

    # 优先按句子边界切分（中英文句号、换行）
    sentences = re.split(r'(?<=[。！？\.\!\?\n])\s*', text)

    current_chunk = ""
    chunk_index = 0

    for sentence in sentences:
        if len(current_chunk) + len(sentence) <= child_chunk_size:
            current_chunk += sentence
        else:
            if current_chunk.strip():
                children.append({
                    "text": current_chunk.strip(),
                    "parent_id": parent.parent_id,
                    "chunk_index": chunk_index,
                    "section_title": parent.title,
                    "filename": parent.filename
                })
                chunk_index += 1

            # 处理超长句子：强制按字符切分
            if len(sentence) > child_chunk_size:
                step = child_chunk_size - child_chunk_overlap
                for i in range(0, len(sentence), step):
                    sub = sentence[i:i + child_chunk_size]
                    if sub.strip():
                        children.append({
                            "text": sub.strip(),
                            "parent_id": parent.parent_id,
                            "chunk_index": chunk_index,
                            "section_title": parent.title,
                            "filename": parent.filename
                        })
                        chunk_index += 1
                current_chunk = ""
            else:
                # 重叠处理：保留上一块的末尾
                if current_chunk and child_chunk_overlap > 0:
                    overlap_text = current_chunk[-child_chunk_overlap:]
                else:
                    overlap_text = ""
                current_chunk = overlap_text + sentence

    # 处理最后一个 chunk
    if current_chunk.strip():
        children.append({
            "text": current_chunk.strip(),
            "parent_id": parent.parent_id,
            "chunk_index": chunk_index,
            "section_title": parent.title,
            "filename": parent.filename
        })

    # 更新父文档的子切片ID列表
    parent.child_chunk_ids = [f"{parent.parent_id}_{c['chunk_index']}" for c in children]

    return children
```

**存储服务**：

```python
# backend/app/services/parent_store.py (新文件)

import logging
from datetime import datetime
from typing import List, Optional
from sqlalchemy.orm import Session
from app.core.config import SessionLocal
from app.models.parent_document import ParentDocumentModel
from .parent_document import ParentDocument  # dataclass

logger = logging.getLogger(__name__)


class ParentDocumentStore:
    """父文档存储服务（SQLite 后端）"""

    def __init__(self):
        self._ensure_table()

    def _ensure_table(self):
        """确保表已创建"""
        from app.core.config import engine, Base
        Base.metadata.create_all(bind=engine)

    def save_parents(self, parents: List[ParentDocument]) -> int:
        """
        批量保存父文档

        策略：先按 filename 删除旧记录，再批量插入（摄入去重）
        """
        if not parents:
            return 0

        db = SessionLocal()
        try:
            # 删除同文件的旧父文档
            filenames = list(set(p.filename for p in parents))
            for fname in filenames:
                db.query(ParentDocumentModel).filter(
                    ParentDocumentModel.filename == fname
                ).delete()

            # 批量插入
            now = datetime.utcnow().isoformat()
            count = 0
            for p in parents:
                model = ParentDocumentModel(
                    parent_id=p.parent_id,
                    filename=p.filename,
                    title=p.title,
                    title_hierarchy=json.dumps(p.title_hierarchy, ensure_ascii=False),
                    content=p.content,
                    section_level=p.section_level,
                    char_count=p.char_count,
                    child_count=len(p.child_chunk_ids),
                    created_at=now,
                    updated_at=now
                )
                db.add(model)
                count += 1

            db.commit()
            logger.info(f"父文档存储完成: {count} 条记录")
            return count

        except Exception as e:
            db.rollback()
            logger.error(f"父文档存储失败: {e}")
            raise
        finally:
            db.close()

    def get_parents_by_ids(self, parent_ids: List[str]) -> List[dict]:
        """批量读取父文档"""
        if not parent_ids:
            return []

        db = SessionLocal()
        try:
            results = db.query(ParentDocumentModel).filter(
                ParentDocumentModel.parent_id.in_(parent_ids)
            ).all()

            return [
                {
                    "parent_id": r.parent_id,
                    "filename": r.filename,
                    "title": r.title,
                    "title_hierarchy": r.get_hierarchy(),
                    "content": r.content,
                    "section_level": r.section_level,
                    "char_count": r.char_count
                }
                for r in results
            ]
        finally:
            db.close()

    def get_parents_by_filename(self, filename: str) -> List[dict]:
        """按文件名读取所有父文档（用于文档预览）"""
        db = SessionLocal()
        try:
            results = db.query(ParentDocumentModel).filter(
                ParentDocumentModel.filename == filename
            ).order_by(ParentDocumentModel.section_level).all()

            return [
                {
                    "parent_id": r.parent_id,
                    "title": r.title,
                    "content": r.content,
                    "char_count": r.char_count
                }
                for r in results
            ]
        finally:
            db.close()

    def delete_by_filename(self, filename: str) -> int:
        """按文件名删除父文档"""
        db = SessionLocal()
        try:
            count = db.query(ParentDocumentModel).filter(
                ParentDocumentModel.filename == filename
            ).delete()
            db.commit()
            return count
        except Exception as e:
            db.rollback()
            logger.error(f"删除父文档失败: {e}")
            raise
        finally:
            db.close()

    def get_stats(self) -> dict:
        """获取父文档存储统计"""
        db = SessionLocal()
        try:
            total = db.query(ParentDocumentModel).count()
            total_chars = db.query(ParentDocumentModel.char_count).all()
            total_chars_sum = sum(c[0] for c in total_chars if c[0]) if total_chars else 0
            return {
                "total_parents": total,
                "total_chars": total_chars_sum,
                "approx_tokens": total_chars_sum // 2  # 中文约2字符/Token
            }
        finally:
            db.close()
```

### 2.3 检索流程重构

#### 2.3.1 父文档检索器

```python
# backend/app/services/parent_retriever.py (新文件)

import logging
from typing import List, Dict
from app.services.vector_store import search_with_distances
from app.services.parent_store import ParentDocumentStore
from app.core.config import get_complete_config

logger = logging.getLogger(__name__)


class ParentDocumentRetriever:
    """
    父文档检索器

    流程:
    1. 向量检索匹配子切片 (child_count=N, 如 20)
    2. 按 parent_id 去重，收集唯一父文档
    3. 按子切片命中数 + 平均距离排序
    4. 提取完整父文档内容
    5. 拼接上下文返回
    """

    def __init__(self):
        self.cfg = get_complete_config()
        self.parent_store = ParentDocumentStore()

        # 可配置参数
        pdr_config = self.cfg.get("parent_document_retrieval", {})
        self.child_retrieve_count = pdr_config.get("child_retrieve_count", 20)
        self.parent_max_count = pdr_config.get("parent_max_count", 5)
        self.parent_max_chars = pdr_config.get("parent_max_chars", 12000)
        self.enable_distance_scoring = pdr_config.get("enable_distance_scoring", True)

    def retrieve(self, query: str) -> List[Dict]:
        """
        执行父文档检索

        Args:
            query: 用户查询

        Returns:
            检索结果列表，每个结果包含:
            - text: 父文档完整内容
            - metadata: 章节标题、文件名、相关度分数等
        """
        # Step 1: 子切片检索
        child_results = search_with_distances(
            query,
            top_k=self.child_retrieve_count
        )

        if not child_results:
            logger.warning("子切片检索无结果")
            return []

        # Step 2: 按 parent_id 聚合
        parent_hits = {}  # parent_id -> {hits, distances, child_texts}
        for child in child_results:
            pid = child["metadata"].get("parent_id")
            if not pid:
                # 兼容遗留数据（无 parent_id 的旧 chunk）
                pid = f"legacy_{child['metadata'].get('source', 'unknown')}"
                if pid not in parent_hits:
                    parent_hits[pid] = {
                        "hits": 0,
                        "distances": [],
                        "child_texts": [],
                        "filename": child["metadata"].get("source", ""),
                        "section_title": child["metadata"].get("section_title", "遗留文档"),
                        "is_legacy": True
                    }

            if pid not in parent_hits:
                parent_hits[pid] = {
                    "hits": 0,
                    "distances": [],
                    "child_texts": [],
                    "filename": child["metadata"].get("source", ""),
                    "section_title": child["metadata"].get("section_title", ""),
                    "is_legacy": False
                }

            parent_hits[pid]["hits"] += 1
            parent_hits[pid]["distances"].append(child.get("distance", 1.0))
            parent_hits[pid]["child_texts"].append(child["text"][:100])

        # Step 3: 排序 —— 命中多 + 距离近的优先
        def score_func(item):
            pid, info = item
            hit_score = info["hits"]
            avg_dist = sum(info["distances"]) / len(info["distances"]) if info["distances"] else 1.0
            # 综合分数：命中数 × (1 - 平均距离)
            return hit_score * (1.0 - avg_dist / 2.0)

        sorted_parents = sorted(parent_hits.items(), key=score_func, reverse=True)
        top_parent_ids = [pid for pid, _ in sorted_parents[:self.parent_max_count]]

        # Step 4: 提取父文档内容
        results = []
        legacy_results = []
        total_chars = 0

        for pid in top_parent_ids:
            info = parent_hits[pid]

            if info["is_legacy"]:
                # 遗留数据：直接使用子切片拼接
                legacy_results.append({
                    "text": "\n\n---\n\n".join(info["child_texts"]),
                    "metadata": {
                        "source": info["filename"],
                        "section_title": "遗留文档",
                        "score": round(score_func((pid, info)), 4),
                        "child_hits": info["hits"],
                        "is_legacy": True
                    }
                })
            else:
                # 新架构：读取完整父文档
                parent_docs = self.parent_store.get_parents_by_ids([pid])
                if parent_docs:
                    doc = parent_docs[0]
                    if total_chars + doc["char_count"] <= self.parent_max_chars:
                        results.append({
                            "text": f"[{doc['title']}]\n{doc['content']}",
                            "metadata": {
                                "source": doc["filename"],
                                "section_title": doc["title"],
                                "title_hierarchy": doc.get("title_hierarchy", []),
                                "score": round(score_func((pid, info)), 4),
                                "child_hits": info["hits"],
                                "char_count": doc["char_count"],
                                "is_legacy": False
                            }
                        })
                        total_chars += doc["char_count"]

        # 遗留结果放在最后
        results.extend(legacy_results)

        logger.info(
            f"父文档检索完成: query='{query[:50]}...' "
            f"→ {len(child_results)} 子切片 → {len(results)} 父文档 "
            f"(总字符: {total_chars})"
        )
        return results
```

### 2.4 摄入流程改造

改造 `ingest_file()` 函数（[ingest.py:287-347](backend/app/services/ingest.py#L287-L347)），支持双路存储：

```python
def ingest_file(path: str, provider: str = "openai", generate_summary: bool = True) -> dict:
    cfg = load_config()
    text = extract_text(path)
    filename = os.path.basename(path)

    result = {
        "filename": filename,
        "chunks_count": 0,
        "parent_count": 0,
        "summary_generated": False,
        "success": False
    }

    # 摄入前去重
    try:
        from app.services.vector_store import delete_documents_by_filename
        deleted = delete_documents_by_filename(filename)
        if deleted > 0:
            logger.info(f"摄入前删除 {filename} 的 {deleted} 个旧文档片段")
    except Exception as e:
        logger.warning(f"摄入前去重删除失败: {e}")

    # 摘要生成（保持不变）
    if generate_summary:
        summary_success = generate_and_save_summary(path, filename, text, provider)
        result["summary_generated"] = summary_success

    # 检查是否启用父文档模式
    pdr_config = cfg.get("parent_document_retrieval", {})
    use_parent_mode = pdr_config.get("enabled", False)

    if use_parent_mode and (filename.endswith('.md') or filename.endswith('.txt')):
        # === 父文档模式 ===
        from app.services.parent_document import generate_parent_documents, generate_child_chunks
        from app.services.parent_store import ParentDocumentStore

        parent_chunk_size = pdr_config.get("parent_max_chars", 8000)
        child_chunk_size = pdr_config.get("child_chunk_size", 300)
        child_chunk_overlap = pdr_config.get("child_chunk_overlap", 60)

        # 第一层：生成父文档
        parents = generate_parent_documents(
            text, filename,
            min_parent_chars=300,
            max_parent_chars=parent_chunk_size
        )
        result["parent_count"] = len(parents)

        # 第二层：生成子切片
        all_children = []
        for parent in parents:
            children = generate_child_chunks(
                parent,
                child_chunk_size=child_chunk_size,
                child_chunk_overlap=child_chunk_overlap
            )
            all_children.extend(children)

        # 存储
        # A. 父文档 → SQLite
        parent_store = ParentDocumentStore()
        parent_store.save_parents(parents)

        # 也删除旧的父文档记录
        parent_store.delete_by_filename(filename)

        # B. 子切片 → ChromaDB
        child_ids = [f"{c['parent_id']}_{c['chunk_index']}" for c in all_children]
        child_texts = [c['text'] for c in all_children]
        child_metas = [
            {
                "source": c['filename'],
                "chunk_index": c['chunk_index'],
                "parent_id": c['parent_id'],
                "section_title": c['section_title'],
                "chunk_type": "child",
                "ingest_time": datetime.datetime.utcnow().isoformat(),
                "file_type": os.path.splitext(path)[1].lower(),
            }
            for c in all_children
        ]

        if child_texts:
            add_documents(ids=child_ids, documents=child_texts, metadatas=child_metas, provider=provider)
            result["chunks_count"] = len(child_texts)
            result["success"] = True
            logger.info(
                f"父文档模式摄入完成: {filename} → "
                f"{len(parents)} 父文档 + {len(child_texts)} 子切片"
            )

    else:
        # === 传统模式（保持不变）===
        chunks = split_text(text, cfg.get("chunk_size", 1500), cfg.get("chunk_overlap", 225))
        metas = []
        ids = []
        for i, c in enumerate(chunks):
            metas.append({
                "source": filename,
                "chunk_index": i,
                "total_chunks": len(chunks),
                "chunk_type": "legacy",  # 标记为遗留数据
                "ingest_time": datetime.datetime.utcnow().isoformat(),
                "file_type": os.path.splitext(path)[1].lower(),
            })
            ids.append(str(uuid4()))
        if chunks:
            add_documents(ids=ids, documents=chunks, metadatas=metas, provider=provider)
            result["chunks_count"] = len(chunks)
            result["success"] = True
            logger.info(f"传统模式摄入完成: {filename} → {len(chunks)} 个文档片段")

    # BM25 索引刷新（两种模式都需要）
    try:
        from app.services.bm25_search import refresh_bm25_index
        refresh_bm25_index()
        logger.info(f"摄入后 BM25 索引已刷新")
    except Exception as e:
        logger.warning(f"摄入后 BM25 索引刷新失败: {e}")

    return result
```

## 四、性能与资源评估

### 4.1 存储增量

| 项目 | 传统模式 | 父文档模式 | 增量 |
|------|---------|-----------|------|
| ChromaDB (子切片) | 1x | ~1.2x (重叠更多) | +20% |
| SQLite (父文档) | - | ~1x (相当于原文) | 新增 |
| 总存储 | 1x | ~2.2x | +120% |

对于 100MB 文档知识库，总存储约 220MB，仍在合理范围。

### 4.2 检索延迟

| 步骤 | 耗时 |
|------|------|
| 子切片向量检索 (top-20) | ~10-30ms |
| parent_id 聚合/排序 | ~1ms |
| SQLite 父文档查询 (IN 查询，有索引) | ~2-5ms |
| **总计增量** | **+3-6ms** |

对整体响应时间（数秒级别）影响可忽略。

### 4.3 内存占用

- `ParentDocumentStore` 无状态服务，每次请求新建 session 即释放
- 父文档内容在检索时按需加载，不常驻内存
- 嵌入模型不变，GPU/内存占用不变

## 五、迁移策略与风险应对

### 5.1 渐进式迁移（推荐）

```
阶段 1 (1-2天): 基础设施搭建
  - 创建 parent_documents 表
  - 实现 ParentDocumentStore
  - 实现 ParentDocumentRetriever
  - 添加配置项，默认 enabled=False
  - 编写单元测试

阶段 2 (2-3天): 灰度验证
  - 选择 2-3 个测试文档用新架构摄入
  - 对比新旧检索结果质量（人工评测 + nDCG）
  - 调整参数（chunk_size、retrieve_count 等）
  - 不改变生产流量

阶段 3 (1天): 全量迁移
  - 清空向量库
  - 重新摄入所有文档（批量脚本）
  - 设置 enabled=True
  - 监控错误率和延迟

阶段 4 (持续): 效果评估
  - 收集用户反馈（👍/👎）
  - 统计引用定位准确率
  - 持续调优参数
```

### 5.2 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| **ChromaDB 维度不匹配**（新旧数据混存） | 检索报错 | 阶段 2 使用独立 collection `lab_docs_v2`；全量迁移后再切换 |
| **父文档过大导致 LLM context 超限** | 回答截断 | `parent_max_chars_total` 硬限制，超过的父文档截断尾部 |
| **遗留文档（PDF/DOCX）不支持** | 旧文档享受不到优化 | PDF 转换后为 Markdown 即可支持；纯文本提取的走传统模式 |
| **SQLite 文件锁**（高并发写入） | 摄入失败 | 摄入本身低频操作（用户手动上传），SQLite WAL 模式足够 |
| **回退场景子切片检索无结果** | 用户看到空回答 | 自动回退到传统混合检索（`fallback_to_hybrid=True`） |

### 5.3 向后兼容

- 传统 chunk 在 metadata 中标记 `chunk_type: "legacy"`
- 父文档检索器对无 `parent_id` 的 chunk 自动归入"遗留"分组
- 传统检索路径完全不受影响（由 `enabled` 开关控制）

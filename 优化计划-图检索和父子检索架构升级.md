# 图检索和父子检索架构升级优化计划

## 概述

本次优化目标是实现更智能的检索流程：**图检索 → LLM 逻辑判断 → 父子检索（带线索）**

---

## 核心设计理念

### 判断标准：逻辑优先，而非数量优先

- ❌ **旧方式**：看有多少个实体、多少个文档（数量判定）
- ✅ **新方式**：看回答问题所需的逻辑链路是否完整（逻辑判定）

### 图构造定位

- ✅ **结构化信息源**：找到关键实体、关系、文档名称
- ❌ **不负责获取详细内容**：详细内容交给父子检索
- ✅ **提供线索给父子检索**：包括目标文档、重点实体、检索角度等

---

## 总体架构

```
用户问题
    ↓
[图检索] → [format_route_result] → 结构化信息 _graph_result
    ↓
[LLM 逻辑判断]
    ├─ 判断标准：逻辑链路是否完整？
    ├─ sufficient=true → [只用图检索结果]
    └─ sufficient=false → [提取线索] → [父子检索（使用线索）]
    ↓
[构建 Context]
    ↓
[LLM 生成回答]
```

### 修改点：图搜索结果格式化

在图搜索模块中增加一个函数，把原始图结果格式化为上述结构，方便 LLM 判断。

**实现函数：`format_route_result(question: str, graph_store, query_router_result: Dict) -> Dict`**

**功能说明**：
- 把原始查询路由器的结果和图存储中的实体/关系数据组合成 LLM 友好的格式

**实现逻辑**：
```python
def format_route_result(question: str, graph_store, router_result: Dict) -> Dict:
    """
    把原始图检索结果格式化为 LLM 判断需要的结构
    
    参数：
        question: 用户问题
        graph_store: GraphStore 实例
        router_result: query_router 返回的原始结果
    
    返回：
        符合规范的 route_result
    """
    route = router_result.get("route", "semantic")
    merged_documents = router_result.get("documents", [])
    
    # 1. 格式化相关实体（包括 properties）
    related_entities = []
    seen_entity_names = set()  # 用于去重
    
    for doc in merged_documents:
        filename = doc.get("filename", "")
        # 从图中查找该文档的实体
        entities = graph_store.get_entities_by_document(filename)
        
        for entity in entities:
            name = entity.get("name")
            if name and name not in seen_entity_names:
                seen_entity_names.add(name)
                related_entities.append(entity)
    
    # 2. 格式化关系路径
    paths = []
    seen_path_keys = set()  # 用于去重路径
    
    # 从问题中提取潜在实体
    entities_from_question = extract_entities_from_question(question)
    
    # 同时加入图中找到的实体，增加路径找到的概率
    entities_from_question.extend([e.get("name") for e in related_entities if e.get("name")])
    
    for entity1 in entities_from_question:
        for entity2 in entities_from_question:
            if entity1 != entity2:
                path = graph_store.find_path(entity1, entity2)
                if path:
                    # 构建路径唯一键，避免重复
                    key = f"{entity1}-{path.get('relation')}-{entity2}"
                    key_rev = f"{entity2}-{path.get('relation')}-{entity1}"
                    
                    if key not in seen_path_keys and key_rev not in seen_path_keys:
                        seen_path_keys.add(key)
                        paths.append({
                            "from": entity1,
                            "relation": path.get("relation", ""),
                            "to": entity2,
                            "description": path.get("description", "")
                        })
    
    return {
        "route": route,
        "related_entities": related_entities,
        "merged_documents": merged_documents,
        "paths": paths
    }
```

```python
def get_entities_by_document(self, filename: str) -> List[Dict]:
    """获取指定文档中提取的所有实体（包括 properties）"""
    doc_node_id = self._make_node_id(filename)
    
    entities = []
    if doc_node_id not in self._graph:
        return entities
    
    # 查找该文档包含的所有实体（CONTAINS关系）
    for neighbor, _, edge_data in self._graph.out_edges(doc_node_id, data=True):
        if edge_data.get("type") == "CONTAINS":
            node_data = self._graph.nodes[neighbor]
            entities.append({
                "name": node_data.get("name", ""),
                "type": node_data.get("type", ""),
                "description": node_data.get("description", ""),
                "properties": node_data.get("properties", {})
            })
    return entities
```

**补充3：`graph_store.py` 新增 `find_path` 方法**
```python
def find_path(self, entity1_name: str, entity2_name: str) -> Optional[Dict[str, str]]:
    """
    在图中查找两个实体之间的直接关系路径
    
    返回：
        如果找到直接关系，返回 { "relation": "...", "description": "..." }
        如果没找到，返回 None
    """
    # 1. 先找到两个实体的节点 ID
    node1_id = None
    node2_id = None
    
    for node_id, node_data in self._graph.nodes(data=True):
        if node_data.get("name") == entity1_name:
            node1_id = node_id
        elif node_data.get("name") == entity2_name:
            node2_id = node_id
    
    if not node1_id or not node2_id:
        return None
    
    # 2. 检查是否有直接关系（两个方向都检查）
    # 方向 1: entity1 -> entity2
    if self._graph.has_edge(node1_id, node2_id):
        edge_data = self._graph.get_edge_data(node1_id, node2_id)
        return {
            "relation": edge_data.get("type", ""),
            "description": edge_data.get("description", "")
        }
    
    # 方向 2: entity2 -> entity1
    if self._graph.has_edge(node2_id, node1_id):
        edge_data = self._graph.get_edge_data(node2_id, node1_id)
        return {
            "relation": edge_data.get("type", ""),
            "description": edge_data.get("description", "")
        }
    
    return None
```

### 第二步：增强父子检索，支持线索传递

#### 修改文件
`backend/app/services/parent_retriever.py`

#### 新增方法

##### 1. `search_in_documents(self, query: str, target_documents: List[str]) -> List[Dict]`

**功能说明：**
- 在指定文档内进行父子检索

**实现逻辑：**
```
1. 初始化 vector store collection
2. 遍历 target_documents 中的每个文档
3. 对每个文档：
   a. 用 query 在该文档内搜索（where={"source": doc_name}）
   b. 获取 top 10 个 child chunks
4. 合并所有文档的 child results
5. 用这些 child chunks 查找对应的 parent documents
6. 返回 parent documents 列表
```

---

##### 2. `enhance_query(self, original_query: str, suggestions: Dict) -> str`

**功能说明：**
- 综合利用线索优化查询

**实现逻辑：**
```
1. 从 suggestions 提取：
   - query_hint
   - focus_entities
   - search_angle
2. 构建 parts 列表：[query_hint]
3. 如果有 focus_entities，加入 parts
4. 如果有 search_angle，加入 parts
5. 去重（小写比对）
6. 用空格连接，返回最终查询
```

---

##### 3. `rerank_with_focus(self, results: List[Dict], focus_entities: List[str]) -> List[Dict]`

**功能说明：**
- 根据关注的实体进行重排序

**实现逻辑：**
```
1. 如果没有 focus_entities，直接返回原结果
2. 遍历每个 result：
   a. 把 text 转小写
   b. 计算提到了多少个 focus_entities
   c. 给 result 增加 focus_match_count 字段
   d. 计算 adjusted_score = original_score + (match_count * 0.2)
3. 按 adjusted_score 降序排序
4. 返回排序后的结果
```

---

##### 4. `_deduplicate_results(self, results: List[Dict]) -> List[Dict]`

**功能说明：**
- 去重结果

**实现逻辑：**
```
1. 用 source 或 parent_id 作为 key 去重
2. 保留分数较高的那个
3. 返回去重后的结果
```

---

##### 5. `retrieve_with_clues(self, original_query: str, suggestions: Dict) -> List[Dict]`

**功能说明：**
- 综合利用所有线索的主检索方法

**实现逻辑：**
```
1. 从配置读取 parent_max_count（如 5）
2. 从 suggestions 提取所有线索
3. 用 enhance_query 优化查询
4. 两阶段检索：
   a. 阶段 1：如果有 target_documents，先在这些文档内搜索
   b. 阶段 2：如果结果 < parent_max_count - 1（如 4），做全库搜索补充
5. 去重
6. rerank_with_focus 重排序
7. 只保留前 N 个（parent_max_count）
8. 返回结果
```
